아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
	        - 단일 책임 원칙(SRP)을 준수할 것
	        - 순수 함수(pure function)로만 구현할 것
	        - 외부 의존성(API, hook, store, router 등)을 참조하지 말 것
	        - 기존 domain 변환 함수들을 그대로 재사용할 것 (중복 로직 작성 금지)
	        - 타입 안정성을 유지하고 any 사용 금지
	        - 파일 간 import 방향(domain → ui 금지)을 준수할 것

==============================================

조건-파일경로) 참고할 파일 경로
	        - src/features/profile/domain/toRadarData.ts
	        - src/features/profile/domain/toActiveTimeText.ts
	        - src/features/profile/domain/toAnimalMeta.ts

조건-파일경로) 구현할 파일 경로
	        - src/viewmodels/profile/toProfileViewModel.ts

==============================================

조건-구현참고) 반드시 참고할 기존 구현 패턴은 다음과 같다.
	        - 각 변환 함수는 Core DTO의 일부 필드만 입력으로 받아 UI 친화적 구조로 변환한다
	        - undefined 입력에 대해 undefined 또는 빈 구조를 반환하며 예외를 발생시키지 않는다
	        - enum → UI meta 변환은 단방향이다
	        - 숫자 계산, 상태 판단, 비즈니스 규칙은 포함하지 않는다

==============================================

핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 각 항목의 적용 여부를 체크리스트로 반환할 것.
	1.	toProfileViewModel 단일 진입 함수 구현
	        - 입력: ProfileCoreDTO
	        - 출력: ProfileViewModel
	        - src/viewmodels/profile/toProfileViewModel.ts에 위치
	2.	내부에서 아래 기존 domain 변환 함수를 조합하여 사용
	        - toRadarData
	        - toActiveTimeText
	        - toAnimalMeta
	3.	ProfileViewModel 구성 규칙
	        - Core DTO를 그대로 노출하지 말 것
	        - UI에서 실제로 사용하는 필드만 포함
	        - optional 필드는 명확히 optional로 유지
	        - 변환 결과가 없는 경우 undefined 반환
	4.	함수 특성
	        - side effect 없음
	        - try/catch 사용 금지
	        - 에러 throw 금지
	        - 상태 판단 로직(온보딩 여부, 완성 여부 등) 포함 금지
	        - API 호출 포함 금지

==============================================

조건-Hook 책임)
	        - hook을 호출하거나 hook과 결합된 로직을 포함하지 않는다
	        - React 의존성(useEffect, useMemo 등)을 전혀 포함하지 않는다

==============================================

조건-UI 책임)
	        - 문자열 i18n 처리 금지
	        - UI 문구 생성 금지
	        - 색상/아이콘 결정은 기존 meta 변환 결과만 사용
	        - ViewModel은 “렌더링 직전 데이터”까지만 책임진다

==============================================

구현 완료 후 반환) 아래 체크리스트를 반드시 반환할 것.
	        - toProfileViewModel 함수가 단일 진입점으로 구현되었는가
	        - ProfileCoreDTO → ProfileViewModel 변환만 수행하는가
	        - 기존 domain 변환 함수들을 직접 재사용했는가
	        - Core DTO 필드를 그대로 노출하지 않았는가
	        - side effect / API / try-catch / throw가 전혀 없는가
	        - 상태 판단 로직이 포함되지 않았는가
	        - 타입 안정성이 유지되었는가
	        - 커서룰(SRP, import 방향, pure function)이 모두 지켜졌는가
