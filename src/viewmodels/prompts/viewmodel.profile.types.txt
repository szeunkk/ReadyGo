핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 각 항목의 적용 여부를 체크리스트로 반환할 것.

1. ProfileViewModel 타입을 신규 정의한다
	        - interface 또는 type alias 중 합리적인 방식을 선택한다

2. Core DTO 필드를 그대로 노출하지 않는다
	        - ProfileCoreDTO 타입을 직접 포함하거나 extends ❌
	        - UI 목적에 맞게 재구성된 필드만 정의

3. UI에서 실제 사용하는 필드만 포함한다
	        - userId
	        - nickname
	        - animalType (optional 여부 명확히)
	        - traits (optional 여부 명확히)
	        - schedule (optional 여부 명확히)

4. optional / required 필드를 명확히 구분한다
	        - UI에서 항상 존재하는 값 vs 조건부 값 구분
	        - undefined / null 중 하나로 일관성 있게 표현

5. 주석으로 타입 책임을 명확히 남긴다
	        - “UI 전용 ViewModel”임을 파일 상단에 명시
	        - Core DTO와의 차이를 간단히 설명
