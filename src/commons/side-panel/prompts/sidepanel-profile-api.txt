핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 각 항목의 적용 여부를 체크리스트로 반환할 것.

1. 인증 / 파라미터
            - Route Handler 내부에서 Supabase 인증 로직을 직접 수행한다
            - supabase.auth.getUser() 결과만을 인증 판단 기준으로 사용한다
	        - 인증되지 않은 요청은 즉시 401로 종료한다
	        - req.params.userId를 사용해 targetUserId를 추출한다
	        - userId 존재 여부만 검증한다 (형식 정규식 검증 X)

2. 권한 체크
	        - 인증된 사용자만 접근 가능하도록 한다
	        - 추가적인 관계/차단/매칭 로직을 절대 포함하지 않는다
	        - 권한 문제 발생 시 403을 반환한다

3. Service 호출
	        - getUserProfileByUserId(targetUserId)를 호출한다
	        - Service 반환값을 가공하지 않고 그대로 사용한다

4. 에러 처리
	        - Service 에러를 HTTP 상태 코드로만 매핑한다
	        - 의미 변환, 메시지 재작성 X

5. 응답
	        - 정상 조회 시 HTTP 200 반환
	        - 응답 body는 ProfileCoreDTO 그대로 반환한다
