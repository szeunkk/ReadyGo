==============================================

조건-네이밍 규칙) 본 프로젝트는 camelCase 네이밍을 표준으로 사용한다.
            - enum key, 객체 key, 속성명은 모두 camelCase를 사용한다.
            - DB에는 enum value(string)만 저장되며, camelCase 값을 사용한다.
            - 한글, SNAKE_CASE, UPPER_SNAKE_CASE 사용 금지.

==============================================

조건-프로젝트 맥락) 본 프로젝트는 "ReadyGo"라는 Steam 기반 게이머 매칭 플랫폼이다.

동물 타입(animalType)은
            - 유저의 플레이 성향을 요약하는 결과 레이블이며
            - 매칭 알고리즘, 프로필 UI, 파티 추천, 필터링, 설명 문구에 공통으로 사용된다.
            - 동물 타입은 점수 계산 대상이 아니다.
            - 성향 점수(traits)는 userTraits에서 계산되며, animalType은 해당 점수의 상대적 분포를 해석한 결과이다.

모든 동물 타입은 dominantTraits, secondaryTraits, avoidTraits를 각각 최소 1개 이상을 반드시 가져야 한다.

==============================================

조건-파일경로) 구현될 TS 파일경로: src/commons/constants/animalType.enum.ts

==============================================

핵심요구사항) ReadyGo에서 사용할 동물 타입(enum) 및 메타데이터를 정의할 것.

핵심요구사항) animalType enum을 string enum 형태로 정의할 것.
            - enum 값은 camelCase 문자열을 사용할 것.
            - 예: wolf, tiger, eagle

핵심요구사항) animalGroup enum을 함께 정의할 것
            - 예: attack, strategy, support, social, lone

핵심요구사항) animalType enum key와 1:1로 매핑되는 메타데이터 객체를 정의할 것.
            - 메타데이터 객체명: animalTypeMeta

핵심요구사항) 각 동물 타입 메타데이터는 다음 정보를 반드시 포함해야 함
            - label: 화면 표시용 한글 이름
            - group: animalGroup 값
            - description: 아래 제공된 기준 설명 문구를 그대로 사용할 것
            - dominantTraits: 두드러지는 성향 키 배열
            - secondaryTraits: 보조 성향 키 배열
            - avoidTraits: 일반적으로 낮은 성향 키 배열
            - ui:
                - image: 내부 svg 이미지 경로

핵심요구사항) 성향(trait)은 수치가 아닌 문자열 key로만 표현할 것.
            - aggression, cooperation, exploration, strategy, leadership, social

핵심요구사항) 메타데이터 객체는 as const로 선언하여 타입 안정성을 유지할 것.

==============================================

동물 타입 목록 및 기준 설명 문구 + 성향 기준표
(총 16종 – 문구 및 성향 기준 수정 금지)

※ 아래 성향 기준은 반드시 다음 규칙으로 trait 배열에 반영할 것:
- "높음"  → dominantTraits
- "중간"  → secondaryTraits
- "낮음" / "매우 낮음" → avoidTraits
