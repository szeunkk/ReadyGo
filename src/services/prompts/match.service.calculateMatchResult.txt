아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
	        - Service Layer는 오케스트레이션만 수행한다
	        - Domain 로직 직접 구현 금지
	        - 계산 정책 / 점수 공식 / factor 정의 금지
	        - Context 조립은 전용 Service에 위임
	        - 단일 public 함수만 export
	        - throw / null 반환 금지
	        - Cold Start 포함 모든 경우에서 결과 DTO 반환
	        - 객체 mutation 금지 (immutability 유지)

==============================================

조건-파일경로) 참고 및 기준 파일 경로는 다음과 같다.
            - src/services/match/calculateMatchResult.service.ts
            - src/services/match/buildMatchContext.service.ts
            - src/domain/match/calculateFinalMatchScore.ts

            - Domain 반환 DTO:
            - src/commons/types/match/matchResultCore.dto.ts

            - Service 반환 DTO:
            - src/commons/types/match/matchResult.dto.ts

==============================================

조건-구현참고) 반드시 참고할 기존 구현 패턴은 다음과 같다.
	        -  buildMatchContext(viewerId, targetUserId)를 통해 MatchContext 생성
	        -  Domain 단일 진입점 calculateFinalMatchScore(context) 호출
            -  calculateFinalMatchScore는 MatchResultCoreDTO를 반환한다
            -  calculateMatchResult.service.ts는 이를 확장하여 MatchResultDTO를 반환한다
            -  Service 내부에서 점수 해석 / 보정 / 조건 분기 없음

==============================================

핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 각 항목의 적용 여부를 체크리스트로 반환할 것.
            1.	calculateMatchResult.service.ts 파일 생성 또는 수정
            2.	단일 public 함수 calculateMatchResult export
            3.	함수 시그니처
                - 입력: (client, viewerId: string, targetUserId: string)
                - 출력: Promise
            4.	buildMatchContext.service.ts를 사용하여 Context 생성
            5.	Context를 직접 조립하거나 수정하지 말 것
            6.	Domain의 calculateFinalMatchScore 단일 함수만 호출
            7.	Domain 내부 factor / sub-function 직접 호출 금지
            8.	Domain 반환값을 재해석하지 말고 그대로 신뢰
            9.	MatchResultCoreDTO를 새 객체로 조립하여 MatchResultDTO로 반환
            10.	UI 문구 / 퍼센트 / 게이지 / 태그 가공 금지
            11.	Object.assign, push, mutation 사용 금지
            12.	throw / null / undefined 반환 금지
	        13.	Cold Start 상황에서도 기본 MatchResultDTO 반환 가능 구조 유지

==============================================

조건-Hook 책임)
            - Hook 또는 UI 로직 호출 ❌
            - 상태 관리 / 캐싱 / 재시도 ❌

==============================================

조건-UI 책임)
            - ViewModel 변환 ❌
            - UI 친화적 문구 생성 ❌
            - 색상 / 게이지 / 텍스트 판단 ❌

==============================================

구현 완료 후 반환) 아래 체크리스트를 반드시 반환할 것.
	        - Service Layer 책임만 수행했는가
	        - 단일 public 함수만 export 했는가
	        - buildMatchContext를 통해 Context를 생성했는가
	        - Context를 직접 조립/수정하지 않았는가
	        - Domain 단일 진입점만 호출했는가
	        - 점수/정책/factor 로직이 Service에 없는가
	        - viewerId + targetUserId 기준으로만 동작하는가
	        - 객체 mutation 없이 새 DTO를 반환하는가
	        - throw / null 반환이 없는가
	        - Cold Start 상황에서도 결과를 반환하는가
	        - 반환 타입이 MatchResultDTO 인가
