조건-구현참고) 반드시 참고할 기존 구현 패턴은 다음과 같다.
	        - Service는 입력 검증 후 Repository를 호출한다
	        - Repository 에러는 ChatFetchError, ChatCreateError, ChatUpdateError, ChatValidationError로 변환한다
	        - Repository는 supabaseAdmin을 직접 사용하므로 Service에서 client 파라미터는 필요 없다
	        - 입력 검증 실패 시 ChatValidationError를 throw한다
	        - Repository 호출 실패 시 적절한 커스텀 에러로 변환하여 throw한다
	        - 반환 타입은 Repository 반환 타입과 동일하게 유지한다
	        - getPartyPost는 null을 반환할 수 있으므로 null 체크 후 적절히 처리한다

핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 각 항목의 적용 여부를 체크리스트로 반환할 것.

1. getPartyPostsService.ts
	        - 함수 시그니처: getPartyPostsService(options?: GetPartyPostsOptions): Promise<PartyPost[]>
	        - 입력 검증: limit은 1 이상, offset은 0 이상, creatorId는 비어있지 않은 문자열(있는 경우), gameTitle은 비어있지 않은 문자열(있는 경우), status는 비어있지 않은 문자열(있는 경우)
	        - 검증 실패 시 ChatValidationError throw
	        - Repository 호출: partyPostsRepository.getPartyPosts
	        - Repository 에러는 ChatFetchError로 변환
	        - 항상 PartyPost[] 배열 반환 (throw ❌ / null ❌ / undefined ❌)

2. getPartyPostService.ts
	        - 함수 시그니처: getPartyPostService(postId: number): Promise<PartyPost | null>
	        - 입력 검증: postId는 양수
	        - 검증 실패 시 ChatValidationError throw
	        - Repository 호출: partyPostsRepository.getPartyPost
	        - Repository 에러는 ChatFetchError로 변환
	        - Repository가 null을 반환할 수 있으므로 null을 그대로 반환 (throw ❌)

3. createPartyPostService.ts
	        - 함수 시그니처: createPartyPostService(input: CreatePartyPostInput): Promise<PartyPost>
	        - 입력 검증: creator_id는 비어있지 않은 문자열, game_title은 비어있지 않은 문자열(trim 후 검증), party_title은 비어있지 않은 문자열(trim 후 검증), start_date는 비어있지 않은 문자열, start_time은 비어있지 않은 문자열, description은 비어있지 않은 문자열(trim 후 검증), max_members는 양수, control_level은 비어있지 않은 문자열, difficulty는 비어있지 않은 문자열
	        - 검증 실패 시 ChatValidationError throw
	        - Repository 호출: partyPostsRepository.createPartyPost (문자열 필드는 trim 처리)
	        - Repository 에러는 ChatCreateError로 변환
	        - 반환 데이터가 없으면 ChatCreateError throw
	        - 항상 PartyPost 객체 반환 (throw ❌ / null ❌ / undefined ❌)

4. updatePartyPostService.ts
	        - 함수 시그니처: updatePartyPostService(postId: number, input: UpdatePartyPostInput): Promise<PartyPost>
	        - 입력 검증: postId는 양수, input의 각 필드가 제공된 경우 해당 필드 검증 (game_title, party_title, description은 trim 후 비어있지 않은 문자열, max_members는 양수, control_level, difficulty는 비어있지 않은 문자열)
	        - 검증 실패 시 ChatValidationError throw
	        - Repository 호출: partyPostsRepository.updatePartyPost (문자열 필드는 trim 처리)
	        - Repository 에러는 ChatUpdateError로 변환
	        - 반환 데이터가 없으면 ChatUpdateError throw
	        - 항상 PartyPost 객체 반환 (throw ❌ / null ❌ / undefined ❌)

5. deletePartyPostService.ts
	        - 함수 시그니처: deletePartyPostService(postId: number): Promise<void>
	        - 입력 검증: postId는 양수
	        - 검증 실패 시 ChatValidationError throw
	        - Repository 호출: partyPostsRepository.deletePartyPost
	        - Repository 에러는 ChatUpdateError로 변환
	        - 반환값 없음 (void)

6. 에러 처리 규칙
	        - ChatValidationError: 입력 검증 실패 (statusCode: 400)
	        - ChatFetchError: 데이터 조회 실패 (statusCode: 500)
	        - ChatCreateError: 데이터 생성 실패 (statusCode: 500)
	        - ChatUpdateError: 데이터 업데이트/삭제 실패 (statusCode: 500)
	        - Repository에서 발생한 에러는 error.message를 originalError로 포함

7. 타입 정의
	        - PartyPost: partyPosts.repository.ts에서 export된 PartyPost 타입
	        - GetPartyPostsOptions: partyPosts.repository.ts에서 export된 GetPartyPostsOptions 인터페이스
	        - CreatePartyPostInput: partyPosts.repository.ts에서 export된 CreatePartyPostInput 인터페이스
	        - UpdatePartyPostInput: partyPosts.repository.ts에서 export된 UpdatePartyPostInput 인터페이스
	        - 에러 타입은 chatErrors.ts에서 import

8. Import 규칙
	        - Repository: import * as partyPostsRepository from '@/repositories/partyPosts.repository'
	        - 에러: import { ChatFetchError, ChatCreateError, ChatUpdateError, ChatValidationError } from '@/commons/errors/chat/chatErrors'
	        - 타입: import type { PartyPost, GetPartyPostsOptions, CreatePartyPostInput, UpdatePartyPostInput } from '@/repositories/partyPosts.repository'
