핵심요구사항)  MCP를 연동하여 피그마의 디자인을 그대로 소스코드로 구현할 것.
핵심요구사항) 다음 조건을 모두 만족하는 완전한 variant 시스템을 구현할 것.
            - state: 'default' | 'hover' | 'filled' | 'error' | 'disabled'
            - theme: 'light' | 'dark'

핵심요구사항) 하드코딩된 색상 및 폰트 값을 절대 사용하지 말 것. 반드시 global.css에 정의된 CSS 변수(상수 토큰)를 사용할 것.

핵심요구사항) selectbox의 Label과 Additional information은 props로 유연하게 넣고 뺄 수 있도록 구현할 것.
            - label?: string | React.ReactNode 형태로 props로 전달받아 표시
            - additionalInfo?: string | React.ReactNode 형태로 props로 전달받아 표시
            - label과 additionalInfo가 모두 있는 케이스로 기본 구현하되, props가 없으면 해당 영역을 표시하지 않도록 조건부 렌더링 구현

핵심요구사항) 셀렉트 박스를 클릭하면 드롭다운으로 옵션 그룹이 나타나도록 구현할 것.
            - selectbox 클릭 시 option-group이 드롭다운 형식으로 표시되도록 구현
            - 드롭다운이 열리고 닫히는 상태 관리 및 애니메이션 구현
            - 외부 클릭 시 드롭다운이 닫히도록 구현

핵심요구사항) 옵션 그룹에서 선택된 옵션은 좌측에 체크표시가 나타나도록 구현할 것.
            - 피그마 디자인을 참고하여 선택된 옵션의 좌측에 체크 아이콘이 표시되도록 구현
            - 선택 상태에 따라 체크 아이콘의 표시/비표시를 조건부 렌더링으로 구현

핵심요구사항) 옵션 아이템의 갯수는 props로 유연하게 관리할 것.
            - items?: Array<{ id: string; value: React.ReactNode; label?: string }> 형태로 props로 전달
            - 아이템 갯수에 따라 동적으로 렌더링되도록 구현
            - 각 아이템 클릭 시 해당 아이템이 선택되도록 상태 관리 구현

핵심요구사항) 스토리북은 구현 금지
