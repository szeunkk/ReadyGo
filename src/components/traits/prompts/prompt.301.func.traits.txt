테스트 작성 원칙) 아래의 작성 원칙을 적용하여, 테스트 코드를 작성할 것.
            - 테스트는 요구사항을 검증하는 명세 역할을 해야한다.
            - 내부 구현 세부사항(private 변수, 내부 함수)에 의존하지 말 것.
            - domain 함수는 순수 함수 테스트로 작성

==============================================

테스트 대상 요구사항) 공통 전제
            - 잘못된 입력이 들어와도 throw 없이 정상 종료되는지 검증
            - undefined/null 반환 금지 케이스 검증
            - 테스트 데이터는 최소/명확/의도 드러나게 구성

테스트 대상 요구사항) 설문 흐름 제어(useTraitSurvey)
            - 초기화 시 질문 리스트가 생성된다.
            - 일반 질문은 랜덤 순서지만 seed 고정 시 항상 동일 순서
            - 스케줄 질문은 항상 마지막 인덱스
            - next 호출 시 인덱스 증가
            - 마지막 질문 이후 completed 상태 전환
            - prev 호출 시 인덱스 감소 (0 이하로 내려가지 않음)
            - reset 호출 시 인덱스 및 completed 상태 초기화 

테스트 대상 요구사항) 답변 상태 관리(useTraitAnswers)
            - 질문 ID 기준으로 답변이 저장된다
            - 동일 질문에 setAnswer 호출 시 overwrite 된다
            - Likert 답변(1~5)이 정상 저장된다
            - Schedule 답변(dayTypes/timeSlots)이 정상 저장된다
            - getAnswer(questionId)로 정확히 조회된다
            - resetAnswer 호출 시 모든 답변이 제거된다
            - 잘못된 타입의 answer 입력 시 상태가 변경되지 않는다

테스트 대상 요구사항) Trait 점수 계산(calculateTraitScores)
            - 답변 값 (1 ~ 5)가 점수(-2 ~ +2)로 정확히 변환된다
            - weight가 점수 계산에 반영된다
            - 동일 axis 질문 점수가 누적된다
            - 답변 없는 질문은 계산에서 제외된다
            - ScheduleAnswer는 점수 계산에서 제외된다
            - 결과 점수가 0~100 범위로 정규화된다
            - axis에 유효 답변이 하나도 없을 경우 중립값(예: 50) 처리 검증
            - 잘못된 answer 타입이 있어도 계산 결과가 깨지지 않는다

테스트 대상 요구사항) 동물 타입 결정(resolveAnimalType)
            - domainant trait 점수가 가장 높은 animal type이 선택된다
            - domainant 동점 시 secondary trait이 tie-breaker로 작동한다
            - avoid trait이 결과에 불리하게 반영된다
            - 동일 점수 조건에서도 항상 동일 결과를 반환한다(결정성)
            - 입력 점수가 비어있거나 일부만 있어도 항상 유효한 animal type이 반환된다

테스트 대상 요구사항) 스케줄 질문 처리
            - 스케줄 질문이 일반 질문과 구분됨을 테스트로 검증
            - dayTypes 다중 선택 구조가 유지됨
            - timeSlots 다중 선택 구조가 유지됨
            - 스케줄 답변이 trait 점수 계산 결과에 영향을 주지 않음을 검증

테스트 대상 요구사항) 예외/실패 케이스 
            - answers가 빈 객체여도 모든 domain 함수가 정상 동작
            - 일부 질문만 답변한 경우에도 계산/결정 가능
            - 잘못된 answer 타입(string, null 등)이 있어도 무시됨
            - 설문 진행 중 reset 호출 시 survey 상태와 answers 상태가 함께 초기화되는 사용 시나리오를 테스트로 검증

구현 완료 후 반환 체크리스트) 구현 완료 후 아래 2가지 체크리스트를 반환할 것.

1) 커서룰(@01-common.mdc, @04-func.mdc) 적용 결과 체크리스트
2) 기능 요구사항 체크리스트
    - 설문 흐름 테스트 
            - 질문 초기화 테스트 (파일/테스트명)
            - 랜덤 + seed 결정성 테스트
            - 스케줄 질문 마지막 위치 테스트
            - next 이동 테스트
            - completed 상태 테스트
            - prev 이동 테스트
            - reset 초기화 테스트

    - 답변 상태 테스트 
            - ID 기준 저장 테스트
            - overwrite 테스트
            - Likert 저장 테스트
            - Schedule 저장 테스트
            - getAnswer 테스트
            - resetAnswers 테스트
            - 잘못된 타입 무시 테스트

    - 점수 계산 테스트
            - 1~5 변환 테스트
            - weight 반영 테스트
            - axis 누적 테스트
            - 무응답 제외 테스트
            - 0~100 정규화 테스트
            - schedule 제외 테스트
            - 빈/잘못된 입력 안정성 테스트

    - 동물 타입 테스트
            - domainant 선택 테스트
            - secondary tie-break 테스트
            - avoid 패널티 테스트
            - 결정성 테스트
            - 항상 유효 반환 테스트

    - 스케줄 처리 테스트
            - 스케줄 타입 구분 테스트
            - 다중 선택 구조 유지 테스트
            - 점수 영향 없음 테스트

    - 예외 시나리도
            - 빈 answer 시나리오
            - 부분 answers 시나리오
            - reset 중 불일치 없음 시나리오

    - 생성된 테스트 파일 목록
            - 주요 describe/it 테스트명 목록
