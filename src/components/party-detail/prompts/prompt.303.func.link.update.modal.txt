핵심요구사항) 다음의 조건에 맞게 모달 열람 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
            1) 모달조건
                - 파티 작성/수정 모달 컴포넌트 경로: src/components/party-submit/partySubmit.tsx
                - 모달 프로바이더 경로: src/commons/providers/modal/modal.provider.tsx
                - 이미 셋팅되어있는 modal.provider를 사용할 것.
                - 이미 셋팅되어있는 modal.provider를 수정하지 말 것.
                - 공통컴포넌트 모달의 커스텀 모달로 구현할 것.
            2) 상세 구현 내용
                - 수정하기 버튼: 클릭시, isEdit=true prop을 전달하여 수정모드로 분기처리하여 모달 컴포넌트를 기존의 페이지 위 중앙에 overlay 하여 겹칠 것.
                - 모달 컴포넌트는 isEdit prop(true/false)으로 수정모드와 작성모드를 분기처리하여 동작할 것.
                    - 수정모드 (isEdit=true): 
                        - party/[id] 페이지의 URL 파라미터에서 id 값을 추출하여 해당 id에 해당하는 게시물의 모든 데이터를 조회할 것.
                        - 조회한 파티 데이터의 모든 필드 값을 수정 컴포넌트의 폼 필드에 초기값으로 설정할 것.
                        - database.types.ts의 party_posts Row 타입에 정의된 모든 필드(control_level, created_at, creator_id, description, difficulty, game_title, id, max_members, party_title, start_date, start_time, tags, voice_chat)가 모달의 기본값으로 모두 설정되어야 함.
                        - 모든 값이 폼에 정확히 매핑되어 표시되어야 함.
                        - 제목을 "파티 수정하기"로 표시하며, API는 PATCH 메서드로 호출할 것.
                    - 작성모드 (isEdit=false): 빈 폼으로 시작하고, 제목을 "새 파티 만들기"로 표시하며, API는 POST 메서드로 호출할 것.
                - 수정모드(isEdit=true)일 경우, party-detail 페이지에서 조회한 파티 데이터의 모든 값을 모달 컴포넌트에 전달할 것.
