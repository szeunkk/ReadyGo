아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
	        - Repository 레이어는 DB 접근 단일 책임만 가진다
	        - 모든 함수는 순수 조회(read-only) 이다
	        - auth, session, RLS, 권한 관련 로직을 포함하지 않는다
	        - 데이터 가공, 기본값 처리, DTO 변환을 절대 하지 않는다
	        - 반환 타입은 DB row 타입 그대로 사용한다
	        - 함수 시그니처만 봐도 책임이 명확해야 한다
	        - Service 레이어에서 바로 조합 가능해야 한다

==============================================

조건-파일경로) 참고 및 기준 파일 경로는 다음과 같다.
	        - src/repositories/userTraits.repository.ts
	        - src/repositories/userProfiles.repository.ts
	        - src/repositories/userPlaySchedules.repository.ts

(해당 파일이 존재하지 않는 경우, 동일 경로 규칙으로 신규 생성한다)

==============================================

조건-구현참고) 반드시 참고할 기존 구현 패턴은 다음과 같다.
	        - 기존 Repository 함수 네이밍 패턴: findByUserId
	        - Supabase client를 사용하는 단순 select + eq 패턴
	        - single() 또는 maybeSingle() 사용 여부는 테이블 특성에 맞게 합리적으로 선택
	        - 에러 throw / 정책 정의 없이, Supabase 응답 구조를 그대로 반환하는 패턴

==============================================

핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 각 항목의 적용 여부를 체크리스트로 반환할 것.

1️. userTraits.repository.findByUserId 구현
	        - 입력: userId: string
	        - 동작: user_traits 테이블에서 user_id 기준 단일 조회
	        - 반환: DB row 원본 데이터 그대로
	        - 가공/변환/기본값 처리 금지

2. userProfiles.repository.findByUserId 구현
	        - 입력: userId: string
	        - 동작: user_profiles 테이블에서 user_id 기준 단일 조회
	        - 반환: DB row 원본 데이터 그대로
	        - nickname, animal_type 등에 대한 fallback 처리 금지

3. userPlaySchedules.repository.findByUserId 구현
	        - 입력: userId: string
	        - 동작: user_play_schedules 테이블에서 user_id 기준 조회
	        - 반환: DB row 또는 row 배열 원본 그대로
	        - null → 빈 배열 변환 금지

4. 공통 규칙 검증
	        - auth 관련 코드가 포함되지 않았는가
	        - Service 로직 없이 Repository 단독 테스트가 가능한가
	        - 함수 이름만으로 책임이 명확한가

==============================================

조건-Hook 책임)
	        - 해당 구현은 Hook을 포함하지 않는다
	        - React / 상태 관리 / effect 로직을 추가하지 않는다

==============================================

조건-UI 책임)
	        - UI 컴포넌트와의 연결 로직을 포함하지 않는다
	        - JSX, styling, props 정의를 포함하지 않는다

==============================================

구현 완료 후 반환) 아래 체크리스트를 반드시 반환할 것.
	        - userTraitsRepository.findByUserId 구현 완료
	        - userProfilesRepository.findByUserId 구현 완료
	        - userPlaySchedulesRepository.findByUserId 구현 완료
	        - userId 기준 단일 조회만 수행함
	        - auth / 권한 관련 로직 미포함
	        - 데이터 가공 및 기본값 처리 미포함
	        - DB row 원본 타입 그대로 반환
	        - Repository 단독 테스트 가능
	        - Service 레이어에서 즉시 조합 가능
