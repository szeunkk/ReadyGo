아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
	        - Route Handler는 인증 + Service 호출 + 에러 매핑만 담당한다
	        - Service 내부 로직을 절대 재구현하지 않는다
	        - userId는 auth.uid()에서만 추출한다 auth.uid() (supabase.auth.getUser()의 user.id)
	        - try/catch는 API(Route) 레벨에서만 허용한다
	        - Service 에러 타입을 의미 그대로 HTTP 응답으로 매핑한다

==============================================

조건-파일경로) 구현할 route handler 파일 경로
            - app/api/profile/me/route.ts

조건-파일경로) 참고할 services 파일 경로:
	        - src/services/profile/getMyProfileService.ts

조건-파일경로) 참고할 profileDTO 파일 경로
	        - src/commons/types/profile/profileCore.dto.ts

조건-파일경로) 참고할 profileErrors 파일 경로
            - src/commons/errors/profile/profileErrors.ts

==============================================

조건-구현참고) 반드시 참고할 기존 구현 패턴은 다음과 같다.
	        - Next.js App Router Route Handler 패턴
	        - supabase.auth.getUser() 기반 인증 처리
	        - Service에서 throw한 커스텀 Error를 API에서 매핑하는 기존 API 구현 패턴

==============================================

핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 각 항목의 적용 여부를 체크리스트로 반환할 것.
	1.	GET /api/profile/me Route Handler 구현
	2.	supabase.auth.getUser()로 인증 처리
	        - 인증 실패 시 401 Unauthorized 반환
	3.	auth.uid()에서 userId 추출
	        - request param / query / body로 userId 받지 않음
	4.	getMyProfileService(authUserId) 호출
	5.	Service 반환값을 그대로 response body로 전달
	6.	Service 에러 매핑
	        - ProfileNotFoundError → 404
	        - ProfileDataInconsistencyError → 500
	        - ProfileFetchError → 500
	7.	정상 응답 시
	        - HTTP 200
	        - 응답 타입: ProfileCoreDTO
	8.	UI 가공, ViewModel 변환, 상태 판단 로직 미포함

==============================================

조건-Hook 책임)
	        - 사용하지 않음

==============================================

조건-UI 책임)
	        - API Route는 UI 책임을 가지지 않는다
	        - 응답 데이터 가공 금지

==============================================

구현 완료 후 반환) 아래 체크리스트를 반드시 반환할 것.
	        - 인증 미들웨어 적용됨
	        - auth.uid()에서 userId 추출함
	        - request param으로 userId 받지 않음
	        - getMyProfileService만 호출함
	        - Service 로직 재구현 없음
	        - Service 에러 타입별 HTTP 매핑 완료
	        - try/catch는 API 레벨에서만 사용됨
	        - 정상 응답 시 ProfileCoreDTO 그대로 반환
	        - UI / ViewModel / 상태 판단 로직 없음
