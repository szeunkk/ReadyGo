조건-공통목록) 사용할 색상 목록(global.css에 명시된 변수 토큰 활용)
            - 색상 토큰은 피그마와 동일하게 color-text-interaction-primary 의 예시와 같이 globals.css의 시멘틱 색상 토큰을 활용하여 구현할 것.
            - 토큰 하드코딩 금지, 반드시 변수 사용.

조건-공통목록) 사용할 타이포그래피 목록(global.css에 명시된 변수 토큰 활용)
            - heading, body, caption 스타일에서 피그마와 유사한 토큰 선택하여 사용.
            - 하드코딩 금지.

조건-공통목록) 사용할 공통 컴포넌트 목록
            - <Avatar /> (사용자 프로필 이미지, size: s, status 표시)
            - <Icon /> (검색, 사용자 메뉴, 전송 버튼 아이콘 등)
            - <Input /> (메시지 입력창, 검색창)

==============================================

핵심요구사항) chat_rooms, chat_room_members, chat_messages, chat_message_reads, chat_blocks ERD 구조를 바탕으로 채팅방 상세 UI만 퍼블리싱할 것(기능 구현 금지).

핵심요구사항) MCP를 연동하여 피그마 디자인의 모든 속성과 변형을 확인하여 해당 컴포넌트를 구현할 것. (기능 구현 X, 퍼블리싱 전용)

            - 채팅방 상세 전체 영역: Implement this design from Figma. https://www.figma.com/design/XCfazIyDI3zOloIscGe6u2/ReadyGo?node-id=2157-1272&m=dev

핵심요구사항) 채팅방 상세 화면은 다음 세 가지 주요 영역으로 구성할 것.
            1) 헤더 영역 (상단 고정)
                - 상대방/그룹 정보: Avatar, 닉네임, 온라인 상태
                - 검색창 (중앙)
                - 사용자 메뉴 버튼 (우측)
            2) 메시지 리스트 영역 (중앙 스크롤)
                - 날짜 구분선 (예: "2025년 12월 12일 금요일")
                - 메시지 버블들 (상대방/본인 구분)
                    - 상대방 메시지: 좌측 정렬, Avatar 포함, 회색 배경
                    - 본인 메시지: 우측 정렬, 초록색 배경
                - 메시지 시간 표시 (각 메시지 옆)
                - 연속된 메시지는 Avatar 중복 표시 최소화
            3) 입력 영역 (하단 고정)
                - 메시지 입력창 (placeholder: "@닉네임 님에게 메시지 보내기")
                - 전송 버튼

핵심요구사항) 각 메시지 버블은 다음 정보를 포함하도록 퍼블리싱할 것.
            1) 메시지 내용 (chat_messages.content)
            2) 메시지 타입 (chat_messages.content_type: text, image, system 등)
            3) 전송 시간 (chat_messages.created_at 포맷팅)
            4) 발신자 정보 (chat_messages.sender_id 기반, 본인/상대방 구분)
            5) 읽음 상태 (chat_message_reads.is_read 기반, 시각적 표현만)

핵심요구사항) 리스트 컨테이너 및 스크롤 스타일은 피그마를 따른다.
            메시지 리스트는 ERD에 있는 필드 정보(room_id, sender_id, content, content_type, created_at 등)를 시각적으로 표현해야 함.

핵심요구사항) 반응형 레이아웃 대응
            - 모바일(≤ 480px): 헤더/입력 영역 padding, 폰트 크기 축소
            - 태블릿/데스크톱: 피그마 기준 사이즈 유지

핵심요구사항) 기능 구현은 하지 말고, 더미 데이터로 UI 상태(메시지 리스트, 발신자 구분 등)를 확인할 수 있도록 Mock 배열 5-10개 정도를 컴포넌트 상단에 작성할 것.

핵심요구사항) 접근성(aria-label 등) 속성을 필수 포함할 것.

핵심요구사항) placeholder 데이터는 ERD 필드명을 기반으로 mock 객체를 선언하여 사용하고, API 연동/상태관리는 구현하지 말 것.

핵심요구사항) 완성된 컴포넌트를 페이지에서 import하여 연결시킬 것.
            - 연결될 경로: src/app/(main)/chat/page.tsx 또는 layout.tsx
            - room이 선택되었을 때 이 컴포넌트를 표시하도록 연결.

세부요구사항) 헤더 영역
            - 배경색: bg-primary 토큰 사용
            - 상대방 정보: Avatar(size: s), 닉네임, 온라인 상태 텍스트
            - 검색창: Input 컴포넌트 또는 동일 스타일, placeholder "검색하기"
            - 사용자 메뉴 버튼: Icon 버튼, 원형 배경

세부요구사항) 메시지 리스트 영역
            - 배경색: bg-secondary 토큰 사용
            - 날짜 구분선: 중앙 정렬, 작은 폰트, 회색 텍스트
            - 메시지 버블 레이아웃:
                - 상대방: 좌측 정렬, Avatar + 메시지 버블 + 시간 (가로 배치)
                - 본인: 우측 정렬, 시간 + 메시지 버블 (가로 배치)
            - 메시지 버블 스타일:
                - 상대방: bg-tertiary, 둥근 모서리
                - 본인: bg-interactive-primary-hover, 검은색 텍스트
            - 연속 메시지 처리: 같은 발신자의 연속 메시지는 Avatar 생략 가능

세부요구사항) 입력 영역
            - 배경색: bg-primary 토큰 사용
            - 입력창: Input 컴포넌트 또는 동일 스타일, placeholder 동적 변경 (상대방 닉네임 포함)
            - 전송 버튼: Icon 버튼, 원형 배경

세부요구사항) 메시지 시간 포맷팅
            - 오늘: "오후 2:30" 형식
            - 어제: "어제 오후 2:30" 형식
            - 그 외: "2025년 12월 12일 금요일" 형식 (날짜 구분선과 동일)

세부요구사항) 스크롤 처리
            - 메시지 리스트는 세로 스크롤 가능
            - 최신 메시지로 자동 스크롤 (기능 구현 X, 스타일만)
