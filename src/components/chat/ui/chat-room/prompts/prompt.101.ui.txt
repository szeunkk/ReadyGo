            - 색상 토큰은 피그마와 동일하게 color-text-interaction-primary 의 예시와 같이 globals.css의 시멘틱 색상 토큰을 활용하여 구현할 것.
            - 타이포그래피 토큰은 heading, body, caption 스타일에서 피그마와 유사한 토큰 선택하여 사용.

            - <Button /> (필요 시)
            - <Icon /> (검색, 사용자 메뉴, 전송 버튼 아이콘 등)
            - <Badge /> 또는 <Tag /> (읽음 상태, 시스템 메시지 등 표시용)
            - <Input /> (메시지 입력창, 검색창)

핵심요구사항) 제공된 ERD(chat_rooms, chat_room_members, chat_messages, chat_message_reads, chat_blocks)를 기반으로 1:1/그룹 공용 chat room 상세 화면을 퍼블리싱할 것.

핵심요구사항) MCP를 연동하여 피그마 디자인의 모든 속성과 변형을 확인하여 해당 컴포넌트를 구현할 것. (기능 구현 X, 퍼블리싱 전용)

            - 채팅방 상세 전체 영역: Implement this design from Figma. https://www.figma.com/design/XCfazIyDI3zOloIscGe6u2/ReadyGo?node-id=2157-1272&m=dev

핵심요구사항) Figma(room-detail) 디자인을 픽셀 단위로 재현하며, UI 퍼블리싱만 수행하고 기능 로직은 작성하지 말 것.

핵심요구사항) 메시지 리스트, 헤더(방 정보/멤버/액션 버튼), 입력 영역(메시지 입력/보내기 버튼), 시스템 Toast/Alert 등 디자인 내 모든 블록을 구현할 것.

핵심요구사항) 채팅방 상세 화면은 다음 세 가지 주요 영역으로 구성할 것.
            1) 헤더 영역 (상단 고정)
                - 상대방/그룹 정보: Avatar, 닉네임, 온라인 상태
                - 검색창 (중앙)
                - 사용자 메뉴 버튼 (우측)
                - 차단된 사용자 상태(chat_blocks)는 헤더 또는 메시지 입력 영역에서 안내 배너로 노출
            2) 메시지 리스트 영역 (중앙 스크롤)
                - 날짜 구분선(Divider) (예: "2025년 12월 12일 금요일")
                - 메시지 버블들 (상대방/본인/시스템 구분)
                    - 상대방 메시지: 좌측 정렬, Avatar 포함, 회색 배경
                    - 본인 메시지: 우측 정렬, 초록색 배경
                    - 시스템 메시지: 중앙 정렬, 특별 스타일
                - 타임스탬프 위치: 각 메시지 옆 또는 하단
                - 연속된 메시지는 Avatar 중복 표시 최소화
                - 스크롤 영역 스타일링
            3) 입력 영역 (하단 고정)
                - 메시지 입력창 (placeholder: "@닉네임 님에게 메시지 보내기")
                - 전송 버튼
                - 입력 영역은 비활성화 상태 스타일까지 퍼블리싱(예: 차단/읽기 전용일 때)

핵심요구사항) 각 메시지 버블은 다음 정보를 포함하도록 퍼블리싱할 것.
            1) 메시지 내용 (chat_messages.content)
            2) 메시지 타입 (chat_messages.content_type: text, image, system 등)
            3) 전송 시간 (chat_messages.created_at 포맷팅)
            4) 발신자 정보 (chat_messages.sender_id 기반, 본인/상대방 구분)
            5) 읽음 상태 (chat_message_reads.is_read 기반, 시각적 표현만)

핵심요구사항) ERD 필드명을 그대로 사용한 mock 데이터 객체를 파일 상단에 선언하여 UI에 바인딩할 것.
            - 예: chat_messages.content_type, chat_message_reads.is_read 등 ERD 필드명 그대로 사용
            - Mock 배열 5-10개 정도를 컴포넌트 상단에 작성

핵심요구사항) 읽음/미읽음, 상대/내 메시지, 시스템 메시지를 시각적으로 구분하고, 각 상태는 globals 토큰으로 스타일링할 것.

핵심요구사항) 접근성(aria-label 등) 속성을 필수 포함할 것.

핵심요구사항) 완성된 컴포넌트를 페이지에서 import하여 연결시킬 것.
            - 연결될 경로: src/app/(main)/chat/@room/page.tsx
            - room이 선택되었을 때 이 컴포넌트를 표시하도록 연결.

세부요구사항) 헤더 영역
            - 배경색: bg-primary 토큰 사용
            - 상대방 정보: Avatar(size: s), 닉네임, 온라인 상태 텍스트
            - 검색창: Input 컴포넌트 또는 동일 스타일, placeholder "검색하기"
            - 사용자 메뉴 버튼: Icon 버튼, 원형 배경
            - 차단된 사용자 안내 배너: chat_blocks 기반 안내 메시지 표시

세부요구사항) 메시지 리스트 영역
            - 배경색: bg-secondary 토큰 사용
            - 날짜 구분선(Divider): 중앙 정렬, 작은 폰트, 회색 텍스트
            - 메시지 버블 레이아웃:
                - 상대방: 좌측 정렬, Avatar + 메시지 버블 + 시간 (가로 배치)
                - 본인: 우측 정렬, 시간 + 메시지 버블 (가로 배치)
                - 시스템: 중앙 정렬, 특별 스타일
            - 메시지 버블 스타일:
                - 상대방: bg-tertiary, 둥근 모서리
                - 본인: bg-interactive-primary-hover, 검은색 텍스트
                - 시스템: 중앙 정렬, 회색 배경 또는 특별 스타일
            - 메시지 bubble tail: 피그마 디자인에 있는 경우 구현
            - 연속 메시지 처리: 같은 발신자의 연속 메시지는 Avatar 생략 가능
            - 스크롤 영역: 세로 스크롤 가능, 스타일링 적용
            - 우측 floating 버튼: 피그마에 있는 경우 구현

세부요구사항) 입력 영역
            - 배경색: bg-primary 토큰 사용
            - 입력창: Input 컴포넌트 또는 동일 스타일, placeholder 동적 변경 (상대방 닉네임 포함)
            - 전송 버튼: Icon 버튼, 원형 배경
            - 비활성화 상태: 차단/읽기 전용일 때 비활성화 스타일 퍼블리싱

세부요구사항) 메시지 시간 포맷팅
            - 오늘: "오후 2:30" 형식
            - 어제: "어제 오후 2:30" 형식
            - 그 외: "2025년 12월 12일 금요일" 형식 (날짜 구분선과 동일)

세부요구사항) UI 디테일
            - hover/focus/pressed 상태: 피그마, 커서룰을 참고하여 구현
            - 메시지 bubble tail: 피그마 디자인에 있는 경우 구현
            - 그림자, 라운드 수치 등은 디자인과 동일하게 globals 토큰으로 표현

세부요구사항) 시스템 Toast/Alert
            - 피그마 디자인에 있는 경우 구현
            - 위치, 스타일, 애니메이션 등 디자인과 동일하게 퍼블리싱
