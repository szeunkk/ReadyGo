핵심요구사항) 각 채팅방 셀은 다음 정보를 포함하도록 퍼블리싱할 것.
            1) 프로필 이미지(Avatar)
            2) 채팅방 이름(1:1 이면 상대방 닉네임, 그룹이면 그룹명)
                - 1:1 방은 상대방 Avatar + 닉네임 중심, 그룹 방은 참여자 수, 대표 Avatar Stack, 그룹 Badge 등으로 구분되게 구성할 것.
            3) 마지막 메시지 미리보기(최대 1줄, 말줄임 처리)
            4) 마지막 메시지 시간(24h, 오늘 내 메시지는 시간, 그 외는 날짜 형식)
            5) 읽지 않은 메시지 수 Badge (0인 경우 미표시)
                - 읽지 않은 메시지가 있는 경우 chat_message_reads.is_read false 상태를 토대로 Unread Badge와 굵은 텍스트 스타일을 적용할 것.
            6) 차단된 유저와의 채팅방은 opacity 0.3 으로 처리 (시각적 표시만, 기능 X)

핵심요구사항) 리스트 컨테이너 및 스크롤 스타일은 피그마를 따른다.
            리스트는 room type, 참여자 정보, 최신 메시지, 읽음 여부, 생성/갱신 시간 등 ERD에 있는 필드 정보를 시각적으로 표현해야 함.

핵심요구사항) 반응형 레이아웃 대응
            - 모바일(≤ 480px): 리스트 항목 padding, 폰트 크기 축소
            - 태블릿/데스크톱: 피그마 기준 사이즈 유지

핵심요구사항) 기능 구현은 하지 말고, 더미 데이터로 UI 상태(읽음/안읽음, 뱃지 수 등)를 확인할 수 있도록 Mock 배열 5개 정도를 컴포넌트 상단에 작성할 것.

핵심요구사항) 접근성(aria-label 등) 속성을 필수 포함할 것.

핵심요구사항) placeholder 데이터는 ERD 필드명을 기반으로 mock 객체를 선언하여 사용하고, API 연동/상태관리는 구현하지 말 것.
