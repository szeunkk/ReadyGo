핵심요구사항) ProfileDropdown 컴포넌트에서 로그아웃 기능을 구현할 것.

1) AuthProvider 연동(필수)
- AuthProvider는 이미 구현되어 있으며, logout 기능을 제공한다:
  - Provider 경로: src/commons/providers/auth/auth.provider.tsx
  - Hook: useAuth()를 통해 logout 함수에 접근 가능
  - logout 함수는 다음을 수행한다:
    - supabase.auth.signOut() 호출 (Supabase 세션 제거)
    - Zustand store clearAuth() 호출 (상태 초기화)
    - URL_PATHS.LOGIN으로 자동 리다이렉트
- ProfileDropdown에서 useAuth hook을 import하여 logout 함수를 사용할 것.
- 기존 AuthProvider 코드는 수정하지 말 것.

2) 로그아웃 버튼 클릭 처리(필수)
- handleLogout 함수를 구현할 것:
  - useAuth hook에서 logout 함수를 가져와 사용
  - logout 함수는 async 함수이므로 await로 호출
  - 로그아웃 후 onClose()를 호출하여 드롭다운 닫기
  - 에러 처리는 logout 함수 내부에서 이미 처리되므로 별도 처리 불필요

3) 코드 수정 위치(필수)
- ProfileDropdown 컴포넌트 내부에서만 수정할 것:
  - useAuth hook import 추가
  - handleLogout 함수 구현 (65-68줄의 TODO 주석 부분)
- 다른 파일은 수정하지 말 것.

==============================================

구현방식)

주의사항)

- AuthProvider의 logout 함수는 이미 모든 로그아웃 로직을 포함하고 있음:
  - Supabase 세션 제거
  - Zustand store 초기화
  - 로그인 페이지로 리다이렉트
- 따라서 ProfileDropdown에서는 logout 함수만 호출하면 됨.
- 추가적인 모달이나 확인 절차는 요구사항에 없으므로 구현하지 말 것.
- onClose()는 로그아웃 후 드롭다운을 닫기 위한 것이므로 반드시 호출할 것.
