핵심요구사항) 다음의 기능을 supabase를 연동하여 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
            1. 조회시나리오
                1-1) 다음의 데이터를 조회할 것.
                        - 접속키: ANON키
                        - 테이블명: user_profiles
                        - 데이터
                            - id: uuid (현재 로그인한 유저의 ID)
                            - nickname: string | null
                            - steam_id: string | null
                        - 조건: 현재 로그인한 유저의 ID로 조회 (단일 레코드)

                1-2) 현재 로그인한 유저의 ID는 useAuth() hook을 통해 가져올 것.
                        - useAuth()에서 제공하는 user.id 사용
                        - user가 null인 경우 에러 처리

                1-3) 조회성공 이후 기존의 mock 데이터를 실제 데이터로 교체할 것.
                        - mock 데이터 위치: profileDropdown.tsx의 39-45번 줄
                        - nickname: 조회된 user_profiles.nickname 사용
                        - isSteamConnected: user_profiles.steam_id가 null이 아니면 true, null이면 false

                1-4) 조회 실패 또는 데이터가 없는 경우 적절한 fallback 처리할 것.
                        - nickname이 null이거나 빈 문자열인 경우: 빈 문자열 또는 기본값 표시
                        - steam_id가 null인 경우: isSteamConnected를 false로 설정
