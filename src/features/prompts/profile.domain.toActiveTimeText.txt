핵심요구사항) 다음 기능을 step-by-step으로 구현하라.

1. 입력 계약 정의
	        - 입력: ProfileCoreDTO.schedule
	        - schedule 타입은 { dayTypes: string[]; timeSlots: string[] } | null | undefined 로 가정한다.

2. schedule 없음 처리
	        - schedule이 undefined 또는 null 인 경우
→ 함수는 반드시 undefined를 반환한다.

3. 텍스트 변환 규칙
	        - dayTypes와 timeSlots를 조합하여 단일 요약 텍스트를 생성한다.
	        - 배열 순서는 입력 그대로 유지한다.
	        - 단순 문자열 조합만 허용한다.
            예:
	        - dayTypes: [“WEEKDAY”, “WEEKEND”]
	        - timeSlots: [“EVENING”]
            → “WEEKDAY, WEEKEND / EVENING”

4. 금지 사항
	        - i18n 문자열 생성 X
	        - 번역 키 변환 X
	        - 의미 해석(예: 평일/주말 설명) X
	        - 기본값 자동 삽입 X
	        - 빈 배열을 임의 문구로 치환 X

5. 반환 타입
	        - 반환 타입은 string | undefined 로 명확히 정의한다.
