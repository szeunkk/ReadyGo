핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 각 항목의 적용 여부를 체크리스트로 반환할 것.

1. 입력 계약
	        - 입력은 ProfileCoreDTO.animalType 하나만 받는다
	        - 입력 타입은 AnimalType | undefined | null 을 허용한다

2. animalType 없음 처리
	        - 입력이 undefined 또는 null 인 경우
	        - 함수는 반드시 undefined 를 반환한다

3. UI 메타 매핑 구현
	        - AnimalType enum 값을 기준으로 UI 메타 정보를 매핑한다
	        - 메타 정보에는 다음 필드를 포함한다
	        - icon
	        - label
	        - 매핑은 Record<AnimalType, AnimalTypeMeta> 형태로 정의한다

4. 변환 규칙
	        - enum → meta 단방향 변환만 수행한다
	        - 조건부 점수, 상태 판단, 해석 로직을 포함하지 않는다

5. 반환 타입
	        - 반환 타입은 AnimalTypeMeta | undefined
	        - UI 컴포넌트에서 바로 사용 가능한 형태여야 한다

구현 완료 후 반환) 아래 체크리스트를 반드시 반환할 것.
